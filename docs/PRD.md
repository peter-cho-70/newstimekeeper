## Newstimekeeper PRD (MVP → 협업 확장)

### 1) 배경 / 문제 정의
- 뉴스 진행 중(라이브 런다운) **편성시간** 내에 전체 뉴스를 마치기 위해, 아이템 수/순서/길이를 수시로 조정해야 한다.
- 현재는 진행 중 조정에 따라 **아래 아이템 시작시간**과 **편성 대비 남은시간(또는 초과)**를 빠르게 재계산/가시화하는 것이 핵심이다.

### 2) 목표(Goals)
- **큐시트(런다운) 기반**으로 아이템 길이/순서를 조정하면 즉시 전체 시간이 재계산되어, 최종적으로 **편성 대비 오차 \(\pm 15초\)** 이내로 마무리하도록 돕는다.
- 프로그램 선택 시(1차: **뉴스외전**, **뉴스와 경제**) 해당 **템플릿**을 불러와 기본 아이템과 기본 길이를 자동 로딩한다.
- MVP는 **웹 앱 + 로컬 단일 사용자(파일 저장/불러오기)**로 제공하고, 이후 협업 기능으로 확장한다.

### 3) 비목표(Non-goals, MVP)
- 방송 송출 제어(플레이아웃), NRCS/뉴스편집시스템 연동, 자동 자막/CG 송출 제어
- 실시간 다중 사용자 동시 편집(2차 로드맵)

### 4) 사용자 / 사용 시나리오
- **사용자**: 뉴스데스크/제작 PD/진행 지원 인력(큐시트 편집 및 진행)

#### 주요 시나리오
- **방송 전**: 프로그램 선택 → 템플릿 자동 로딩 → 편성시간/뉴스시작시간 확인 → 아이템(완제/단신/기자/제목/길이/비고) 편집 → 저장
- **진행 중**: 아이템 진행 흐름에 따라 아이템 길이 조정(10/15/20초 버튼, 또는 수동 입력) / 아이템 순서 조정 / 아이템 제외(뉴스끝 아래로 이동) → 남은시간이 목표 범위에 들어오도록 맞춤
- **방송 후**: 최종 큐시트를 파일로 저장(기록/재사용)

### 5) 성공 지표(Success Metrics)
- 진행 중 편집 시 **재계산 반영 지연**이 체감상 즉시(\(<100ms\) 목표)로 느껴짐
- 방송 종료 시 편성 대비 오차가 **\(\pm 15초\)** 이내로 들어가는 빈도 증가(현장 피드백 기반)

## 6) 화면 / IA (UI 레퍼런스 반영)
사용자가 제공한 기존 뉴스진행 화면을 참고해 **어두운 테마(다크 UI)**, **상단 요약바 + 아이템 테이블** 구조를 기본으로 한다.

### 6.1 프로그램 선택 화면
- **프로그램 목록**: `뉴스외전`, `뉴스와 경제`
- 선택 즉시 해당 템플릿을 로딩하여 큐시트 화면으로 이동

### 6.2 큐시트 편집/진행 화면
#### A) 상단 요약바(Sticky)
- **프로그램명**, **방송일자**
- **뉴스 시작시간**(HH:MM:SS)
- **편성시간**(mm:ss 또는 HH:MM:SS)
- **진행시간**(타이머, 00:00:00)
- **잔여시간(편성 대비)**: 색상 규칙
  - **여유(+): 녹색**
  - **초과(-): 빨간색**
- (선택) **PD/담당** 등의 메타 표시
- (선택) `다시읽기/리로드` 버튼

#### B) 아이템 테이블
- 1행=1아이템
- 중간에 **빈 줄(구분용)** 삽입 가능
- 섹션/블록 헤더(예: “오프닝”, “앵커 블록”)도 행으로 표현 가능(계산에는 기본 제외)

##### 컬럼(초안)
- **순서**: 표시 및 빠른 이동(위/아래)
- **구분**: 완제/단신/기타(드롭다운)
- **기자(담당)**: 텍스트
- **제목(아이템)**: 텍스트
- **길이**: mm:ss 표기 + 조정 UI
- **상태 플래그**: (기존 UI의 V/A/C 등) MVP에서는 “태그”로 표현하고 의미는 설정에서 확장 가능
- **비고**: 텍스트

##### 행 액션(필수)
- **길이 빠른 조정**: +10 / +15 / +20초, -10 / -15 / -20초
- **길이 수동 입력**: mm:ss 또는 초 단위 입력(검증/반올림 규칙 적용)
- **위로/아래로 이동**: 1줄씩 이동(키보드 단축키 지원 권장)
- **삭제**
- **진행 제외**: `뉴스끝` 아래로 이동하거나, 행 단위 제외 토글(둘 중 하나는 MVP에 포함)

#### C) 뉴스끝(뉴스 종료 라인)
- 테이블 내 **고정 표시 행**으로 존재
- `뉴스끝` 아래의 아이템은 기본적으로 **진행 계산에서 제외**
- 사용자는 아이템을 `뉴스끝` 위/아래로 이동하여 포함/제외를 제어한다

## 7) 기능 요구사항
### 7.1 템플릿 로딩
- 프로그램 선택 시 해당 템플릿을 불러온다.
- 템플릿은 다음을 포함한다.
  - 기본 아이템 목록(오프닝/타이틀/광고/앵커 인사/클로징 등)
  - 각 아이템의 기본 길이
  - **기본 아이템 여부**(예: 배지/잠금/리셋 기준)

#### 프로그램별 템플릿(초안)
- `뉴스외전`: 예시 템플릿 파일을 제공하고, 실제 기본 아이템/기본 길이는 운영 정의로 교체한다.
- `뉴스와 경제`: 예시 템플릿 파일을 제공하고, 실제 기본 아이템/기본 길이는 운영 정의로 교체한다.

참고: 예시 파일
- `docs/templates/news_extra.template.example.json`
- `docs/templates/news_economy.template.example.json`

### 7.2 시간 자동 계산(핵심)
- `뉴스 시작시간`과 `뉴스끝 위`의 “진행 포함” 아이템들의 길이 합으로,
  - 각 아이템의 **시작 시각**(=이전 아이템 시작 + 이전 길이)을 계산한다.
  - 전체 합계가 편성시간을 넘는지/남는지 계산하여 상단에 표시한다.
- 아이템 길이/순서/포함여부/빈줄 여부가 바뀌면 즉시 재계산한다.

### 7.3 길이 조정 UX
- 버튼 증감은 **10/15/20초 단위**를 제공한다(누적 가능).
- 수동 입력을 허용한다.
  - 입력 포맷: 기본 `mm:ss` (선호), 보조로 `초` 입력 지원 가능
  - 검증: 음수 금지, 상한(예: 59:59) 설정 가능
  - 반올림: 버튼 단위와 충돌하지 않도록, 수동 입력은 **그대로** 저장(기본)하되 설정으로 “가까운 5초/10초 반올림” 옵션을 둘 수 있다.

### 7.4 아이템 편집
- 구분/기자/제목/비고를 실시간 편집
- 빈 줄 추가/삭제
- 섹션 헤더(오프닝 등) 행을 추가할 수 있음(기본: 계산 제외)

### 7.5 로컬 저장/불러오기(MVP)
- 현재 큐시트를 파일로 **내보내기/불러오기**
- 템플릿도 파일로 관리 가능(초기 배포는 앱 번들 내 기본 템플릿 + 사용자 덮어쓰기)

## 8) 시간/데이터 계산 규칙(정의)
### 8.1 단위/표기
- 내부 저장 단위: **초(정수)** 권장
- 표시: `mm:ss` (60분 이상은 `HH:MM:SS` 확장)

### 8.2 포함/제외 규칙
- 기본 포함: `뉴스끝` 위의 아이템 중 `includeInRun=true`인 행
- 기본 제외:
  - `뉴스끝` 아래
  - `blank`(빈 줄)
  - `sectionHeader`(섹션 헤더)

### 8.3 시작시각 계산
- `itemStartTime[0] = newsStartTime`
- `itemStartTime[i] = itemStartTime[i-1] + durationSeconds[i-1]` (포함되는 행 기준으로만 누적)

### 8.4 편성 대비 남은시간
- `delta = scheduledSeconds - sum(includedDurations)`
  - delta > 0: **남음(+)** → 녹색
  - delta = 0: 중립
  - delta < 0: **초과(-)** → 빨간색

### 8.5 허용 오차 목표
- 화면 상단에 “편성대비 오차(초)” 또는 “잔여/초과”를 표시
- 목표: **\(\pm 15초\)** 내로 종료하도록 운영 지원

## 10) 비기능 요구사항
- **성능**: 500행 기준 편집/재계산 즉시 반영 체감(가상 스크롤 고려)
- **안정성**: 브라우저 새로고침/탭 종료 대비 자동 임시저장(로컬 스토리지) 옵션
- **접근성/가독성**: 다크 테마, 숫자(시간) 가독성 최우선(큰 폰트/높은 대비)

## 11) 범위/릴리즈 계획
### MVP (웹 + 로컬 단일 사용자)
- 프로그램 선택(2개)
- 템플릿 로딩
- 아이템 편집/삭제/순서 변경
- 10/15/20초 증감 + 수동 입력
- 뉴스끝 라인 + 포함/제외
- 로컬 파일 저장/불러오기(JSON)
